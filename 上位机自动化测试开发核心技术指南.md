## 一、通信与硬件交互层

### 1.1 Serial (串口通信)

**核心库**: `pyserial`

**重点知识点**:
- **timeout 配置策略**: 阻塞 vs 非阻塞模式选择
- **硬件流控**: `rtscts`, `dsrdtr` 参数配置
- **虚拟串口调试技巧**: 使用 `socat` 或 `com0com` 创建虚拟端口对

**避坑指南**:
- ❌ 慎用 `readline()` - 依赖换行符，若设备不发换行符会死锁
- ⚠️ 拔出 USB 转串口线时，`read()` 可能不抛异常而是返回空，需配合 `isOpen()` 检查
- ✅ 推荐使用 `read(size)` + 超时机制，或使用线程异步读取

```python
import serial

ser = serial.Serial(
    port='/dev/ttyUSB0',
    baudrate=115200,
    timeout=1.0,  # 1秒超时
    rtscts=False,  # 禁用硬件流控
    dsrdtr=False
)

# 安全读取示例
def safe_read(ser, size=1024):
    if not ser.is_open:
        return None
    try:
        data = ser.read(size)
        return data if data else None
    except serial.SerialException as e:
        print(f"Serial error: {e}")
        return None
```

### 1.2 TCP/UDP (网络通信)

**核心库**: 原生 `socket`

**重点知识点**:
- **TCP_NODELAY**: 禁用 Nagle 算法，减少延迟（上位机实时性要求）
- **粘包/半包解析**: 长度前缀法（4字节大端长度 + 数据体）
- **心跳与自动重连机制**: 应用层心跳保活 + 指数退避重连

**避坑指南**:
- ❌ 避免在主线程调用 `connect()`
- ⚠️ UDP 注意丢包与乱序，不适合传输关键控制指令
- ✅ 设置合理的接收超时，避免永久阻塞

```python
import socket
import struct

class TcpClient:
    def __init__(self, host, port):
        self.sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.sock.setsockopt(socket.IPPROTO_TCP, socket.TCP_NODELAY, 1)
        self.buffer = bytearray()
        
    def connect(self, host, port):
        self.sock.connect((host, port))
        self.sock.settimeout(1.0)  # 短超时配合循环解析
        
    def send_frame(self, payload: bytes):
        # 长度前缀协议：4字节大端长度 + 数据
        header = struct.pack(">I", len(payload))
        self.sock.sendall(header + payload)
        
    def recv_frames(self):
        chunk = self.sock.recv(4096)
        if not chunk:
            return []
        self.buffer.extend(chunk)
        
        frames = []
        while len(self.buffer) >= 4:
            length = struct.unpack(">I", self.buffer[:4])[0]
            if len(self.buffer) < 4 + length:
                break
            frames.append(self.buffer[4:4+length])
            self.buffer = self.buffer[4+length:]
        return frames
```

**适用场景**: 大多数 I/O 密集型任务（等待串口数据、网络通信）

from PySide6.QtCore import QObject, QThread, Signal

class CommunicationWorker(QObject):
    data_received = Signal(bytes)
    connection_lost = Signal()
    
    def __init__(self, serial_port):
        super().__init__()
        self.serial_port = serial_port
        self._running = False
    
    def start_communication(self):
        self._running = True
        while self._running:
            data = self.serial_port.read(1024)
            if data:
                self.data_received.emit(data)
            else:
                time.sleep(0.01)
    
    def stop(self):
        self._running = False

# 使用方式
worker = CommunicationWorker(serial_port)
thread = QThread()
worker.moveToThread(thread)
thread.started.connect(worker.start_communication)
worker.data_received.connect(handle_data)

## 六、TCP/UDP通信可靠性设计

### 6.1 粘包/半包处理

**必须使用应用层协议定义消息边界**

**推荐方案**: 长度前缀法（4字节大端长度 + 数据体）

```python
def parse_frames(buffer: bytearray) -> tuple:
    """解析完整帧，返回 (帧列表, 剩余缓冲区)"""
    frames = []
    while len(buffer) >= 4:
        length = struct.unpack(">I", buffer[:4])[0]
        if length > 10 * 1024 * 1024:  # 限制 10MB
            buffer.clear()
            raise ValueError("Invalid packet length")
        
        if len(buffer) >= 4 + length:
            frame = buffer[4:4+length]
            buffer = buffer[4+length:]
            frames.append(frame)
        else:
            break  # 包体不完整，等待下一次 recv
    
    return frames, buffer
```

### 6.2 心跳机制

**应用层心跳用于**:
- 快速检测设备是否掉线
- 保持 NAT/防火墙映射不被回收
- 作为系统空闲时的保活信号

```python
import time

class HeartbeatManager:
    def __init__(self, interval=5.0, timeout=15.0):
        self.interval = interval
        self.timeout = timeout
        self.last_heartbeat_time = time.time()
    
    def should_send_heartbeat(self) -> bool:
        return time.time() - self.last_heartbeat_time >= self.interval
    
    def check_timeout(self) -> bool:
        return time.time() - self.last_heartbeat_time >= self.timeout
    
    def mark_active(self):
        self.last_heartbeat_time = time.time()
```

### 6.3 超时与重连

```python
import time

class AutoReconnectClient:
    def __init__(self, max_retries=5, base_delay=1.0):
        self.max_retries = max_retries
        self.base_delay = base_delay
    
    def connect_with_retry(self):
        for attempt in range(self.max_retries):
            try:
                self.connect()
                return True
            except Exception as e:
                delay = self.base_delay * (2 ** attempt)  # 指数退避
                print(f"Retry {attempt + 1}/{self.max_retries} after {delay}s")
                time.sleep(delay)
        return False
```

## 七、工业通信协议选型原则

### 7.1 为什么优先使用原始 socket 而非 telnetlib？

| 维度 | telnetlib | socket |
|:---|:---|:---|
| **数据透明度** | 自动过滤/注入 Telnet 控制字符，原始字节流被修改 | 完全透传，收发内容与线上一致 |
| **粘包/拆包控制** | 封装了内部缓冲，难以精细控制 | 可直接操作 `recv()` 返回的 chunk |
| **心跳机制** | 无内置心跳，需手动发包且可能被选项协商干扰 | 可自由设计应用层心跳包格式与超时逻辑 |
| **二进制协议支持** | 针对终端交互优化，处理非 ASCII 数据易出错 | 原生 bytes 支持，配合 struct 完美适配工业协议 |
| **Python 版本状态** | Python 3.11+ 标记为 deprecated | 核心标准库，长期维护 |
| **实时性控制** | 内部轮询/缓冲机制导致延迟不可控 | 可设置 `TCP_NODELAY`、自定义超时 |

**结论**: 
- ✅ 如果设备协议是 `0xAA 0x55 [LEN] [DATA] [CRC]` 这种二进制格式，**必须用 socket**
- ✅ 如果设备显示 `Welcome to Device A. Login:` 这类交互式终端，**可使用 pexpect**
- ❌ 仅在对接标准 Telnet 终端设备且无需高可靠数据传输时，才可短期使用 `telnetlib`

### 7.2 pexpect 的使用场景

**核心定位**: 模拟人类与命令行/交互程序进行对话

**典型场景**:
1. 自动化交互命令行工具（iperf、ping、openocd）
2. 串口终端自动化（替代复杂的 pyserial 状态机）
3. SSH/Telnet 远程登录（处理各种非标准提示符）

```python
import pexpect

def ssh_login(host, user, password):
    child = pexpect.spawn(f'ssh {user}@{host}')
    i = child.expect(['Are you sure.*', 'password:', pexpect.EOF])
    
    if i == 0:
        child.sendline('yes')
        child.expect('password:')
    
    child.sendline(password)
    i = child.expect(['#', '$', 'Permission denied'])
    
    if i == 2:
        return False
    
    child.sendline('ls -l')
    child.expect('#')
    print(child.before.decode())
    child.close()
    return True
```

### 7.3 TCP/IP 协议详解

**协议概述**: TCP/IP是互联网的核心协议套件，包括TCP（传输控制协议）和IP（网际协议）。上位机开发中主要使用TCP进行可靠的数据传输。

**核心特性**:
- **面向连接**: 三次握手建立连接，四次挥手断开连接
- **可靠传输**: 序号确认、重传机制、流量控制、拥塞控制
- **字节流服务**: 无边界的字节序列，需应用层定义消息边界

**TCP_NODELAY 优化**:
```python
import socket

sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
sock.setsockopt(socket.IPPROTO_TCP, socket.TCP_NODELAY, 1)  # 禁用Nagle算法
```

**应用场景**:
- ✅ 设备远程监控与控制
- ✅ 大数据量可靠传输
- ✅ 需要确认机制的指令下发

### 7.4 Modbus 协议详解

**协议概述**: Modbus是工业领域最流行的串行通信协议之一，用于主从设备间的通信。

**主要变体**:
| 变体 | 物理层 | 传输方式 | 特点 |
|:---|:---|:---|:---|
| **Modbus RTU** | RS232/RS485 | 二进制 | 高效、抗干扰强 |
| **Modbus ASCII** | RS232/RS485 | ASCII文本 | 调试方便、效率低 |
| **Modbus TCP** | Ethernet | TCP/IP | 网络传输、跨网段 |

**数据帧结构 (RTU模式)**:
```
[设备地址(1B)] [功能码(1B)] [数据(NB)] [CRC校验(2B)]
```

**功能码说明**:
| 功能码 | 名称 | 用途 |
|:---|:---|:---|
| 0x03 | 读取保持寄存器 | 读取模拟量输出 |
| 0x04 | 读取输入寄存器 | 读取模拟量输入 |
| 0x05 | 强制单线圈 | 控制单个继电器 |
| 0x06 | 预置单寄存器 | 设置单个寄存器值 |

**Python实现示例**:
```python
from pymodbus.client import ModbusTcpClient

client = ModbusTcpClient('192.168.1.100', port=502)
client.connect()

# 读取保持寄存器（地址0开始，读取10个）
result = client.read_holding_registers(address=0, count=10, slave=1)
if not result.isError():
    print(f"寄存器值: {result.registers}")

# 写入单个寄存器
client.write_register(address=0, value=1234, slave=1)

client.close()
```

**避坑指南**:
- ⚠️ RTU模式下需严格遵守从站地址和CRC校验
- ⚠️ TCP模式需处理连接断开重连
- ✅ 使用功能码0x10进行多寄存器批量写入更高效

### 7.5 CAN/CAN-FD 协议详解

**协议概述**: CAN（Controller Area Network）是一种串行通信协议，最初为汽车电子设计，现已广泛应用于工业控制。

**核心特性**:
- **多主通信**: 无主从之分，任意节点可主动发送
- **优先级仲裁**: 基于ID的非破坏性仲裁
- **错误检测与处理**: 位错误检测、ACK确认、自动重传

**CAN 2.0A/B vs CAN-FD**:
| 特性 | CAN 2.0 | CAN-FD |
|:---|:---|:---|
| 数据长度 | 最大8字节 | 最大64字节 |
| 波特率 | 最高1Mbps | 数据段最高8Mbps |
| 标识符 | 11位(2.0A)/29位(2.0B) | 扩展帧支持 |

**DBC文件解析**:
```python
import can
import cantools

db = cantools.database.load_file('vehicle.dbc')
bus = can.interface.Bus(channel='can0', bustype='socketcan')

# 发送报文
msg = db.get_message_by_name('EngineData')
data = msg.encode({'RPM': 3000, 'Temperature': 85.5})
can_msg = can.Message(arbitration_id=msg.frame_id, data=data, is_extended_id=False)
bus.send(can_msg)

# 接收解析
for rx_msg in bus:
    try:
        decoded = db.decode_message(rx_msg.arbitration_id, rx_msg.data)
        print(decoded)
    except:
        pass
```

**应用场景**:
- ✅ 车载电子控制系统
- ✅ 工业自动化设备
- ✅ 机器人关节控制

### 7.6 OPC UA 协议详解

**协议概述**: OPC UA（Open Platform Communications Unified Architecture）是工业4.0时代的标准通信协议，支持跨平台、跨厂商的互操作。

**核心优势**:
- **平台无关性**: 支持Windows/Linux/嵌入式系统
- **安全性**: 内置加密、认证、授权机制
- **信息建模**: 统一的地址空间和数据模型
- **可靠性**: 断线重连、数据缓存

**架构层次**:
```
┌─────────────────────────────────────────┐
│            应用层 (Application)          │
├─────────────────────────────────────────┤
│            服务层 (Services)             │
├─────────────────────────────────────────┤
│         安全层 (Security)               │
├─────────────────────────────────────────┤
│       传输层 (Transport)                │
│    (TCP/UDP/HTTP/WebSocket)            │
└─────────────────────────────────────────┘
```

**Python实现示例**:
```python
from opcua import Client

client = Client("opc.tcp://localhost:4840/freeopcua/server/")
try:
    client.connect()
    
    # 获取节点
    root = client.get_root_node()
    objects = client.get_objects_node()
    
    # 读取变量
    temp_node = client.get_node("ns=2;i=2")
    temp_value = temp_node.get_value()
    print(f"温度: {temp_value}")
    
    # 写入变量
    setpoint_node = client.get_node("ns=2;i=3")
    setpoint_node.set_value(25.0)
    
finally:
    client.disconnect()
```

**信息模型**:
- **节点(Node)**: 地址空间的基本单元
- **对象(Object)**: 表示设备或系统组件
- **变量(Variable)**: 存储数据值
- **方法(Method)**: 可调用的操作

**应用场景**:
- ✅ 工业4.0智能工厂
- ✅ 跨厂商设备集成
- ✅ 云端数据采集与监控

#### 一、串行通信

| 协议 | 物理接口 | 传输距离 | 特点 | 应用场景 |
|:---|:---|:---|:---|:---|
| **RS232** | DB9 | ~15米 | 点对点、全双工 | 设备调试、控制台 |
| **RS485** | A/B差分 | ~1200米 | 多点通信、半双工 | 工业现场总线 |

#### 二、网络通信

| 协议 | 特点 | 端口 | 安全性 |

| **UDP** | 无连接、快速 | 自定义 | 低 |
| **Telnet** | 明文终端协议 | 23 | ❌ 不安全 |

| **HTTP/HTTPS** | Web协议 | 80/443 | HTTPS安全 |
| **MQTT** | 轻量发布/订阅 | 1883/8883 | 可加密 |

**Telnet vs SSH**: 
- Telnet: 明文传输，已被淘汰
